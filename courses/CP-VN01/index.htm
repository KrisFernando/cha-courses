Virtual Networks

Objective: This course is designed to introduce students to the fundamentals of virtual networks or networking in cloud computing. You will learn about the various components and services offered by cloud providers to facilitate all your networking needs.


Author and Designer: Kris Fernando

Introduction

Virtual networks in the cloud are isolated network environments that allow you to securely connect and manage your cloud resources. They provide a private space within a public cloud infrastructure, enabling you to create subnets, configure routing rules, and control network traffic. Virtual networks are essential for building scalable and secure cloud applications, as they provide a foundation for connecting your resources and ensuring data privacy.

Classless Inter-Domain Routing (CIDR)

An IPv4 (Internet Protocol version 4) address is a 32-bit number that uniquely identifies a device on a network.

It's typically written in dotted-decimal notation, consisting of four decimal numbers separated by periods. Each decimal number, ranging from 0 to 255, represents an octet (8 bits) of the address.

IPv4 CIDR Notation

In IPv4, CIDR notation is represented as follows:

  • IP address: The specific address within the network.
  • Slash (/): Separates the IP address from the prefix length.
  • Prefix length: The number of bits used for the network portion of the IP address.
Example:

192.168.1.0/24: This represents a network with a prefix length of 24 bits. The first 24 bits (192.168.1.0) define the network address, and the remaining 8 bits (0-255) are available for host addresses.

Summary
  • CIDR provides a flexible and efficient way to allocate IP addresses.
  • By understanding CIDR notation and how to calculate network sizes, you can effectively design and manage IP address spaces for your virtual networks.

Virtual Private Cloud (VPC)

  • A VPC is a virtual network that provides an isolated environment for your cloud resources within a specific region.
  • It allows you to have private IP address spaces, control security settings, and manage network traffic.
  • VPCs can be customized to meet your specific networking requirements.
Consider the following for VPCs:
  • Region: The geographic location where the VPC will be created.
  • CIDR block: The range of IP addresses that will be used within the VPC.
  • Tenancy: Whether the VPC will be default (shared) or dedicated.

Subnets

Sizing and Subnetting:
  • Network Size: The number of hosts in a network is calculated as 2^(32 - prefix length) - 2. The two addresses subtracted are the network address itself and the broadcast address.
  • Subnetting: Dividing a larger network into smaller subnetworks is called subnetting. This is done by borrowing bits from the host portion of the IP address to create additional network bits.
Example:

10.0.0.0/16: This network can accommodate 65,534 hosts. If we want to create three subnets, we can borrow eight bits from the host portion, resulting in a prefix length of 24:

  • 10.0.0.0/24: 1st subnet with 254 hosts
  • 10.0.1.0/24: 2nd subnet with 254 hosts
  • 10.0.2.0/24: 3rd subnet with 254 hosts
  • We have additional space to create subnets for future use.
Private or Public:
  • Private Subnet: does not have inbound traffic originating from the public internet.
  • Public Subnet: has inbound traffic originating from the public internet.

Route Tables

Route tables serve as the instruction manuals for routers. They provide a detailed roadmap, outlining the best paths for network traffic to reach its destination. Each route within a table specifies the destination network, the next hop to reach that destination, and the interface to use. Routers consult these tables to determine the most efficient way to forward packets, ensuring that data flows smoothly across the network.

Understanding Route Tables
  • Route tables are used to determine the path that network traffic should take to reach a specific destination.
  • They contain a set of rules, called routes, that specify the destination network, the next hop to reach that destination, and the interface to use.
  • Route tables are essential for ensuring that network traffic is routed correctly within a VPC.
Creating Route Tables
  • To create a route table, you need to specify the VPC it will belong to.
  • You can then add routes to the table to define the paths for different destinations.
  • Route tables are then associated to subnets so that hosts within that subnet have clear routes.
Route Table Components

Route tables consist of the following components:

  • Destination CIDR block: The network or subnet that the route applies to.
  • Target: The next hop to reach the destination, which can be a gateway, another route table, or an virtual machine.

Internet Gateways

Understanding Internet Gateways (IGW)
  • An internet gateway is a network appliance that provides connectivity between a VPC and the public internet.
  • It allows virtual machines within the VPC to access resources on the internet and receive incoming traffic from the internet.
  • Internet gateways are essential for enabling communication between your VPC and the outside world.
Using an Internet Gateway
  1. Create an Internet Gateway (IGW).
  2. Attach it to a Virtual Private Cloud (VPC).
  3. Create a route in a Route Table to the IGW.
  4. Associate the Route Table to a Subnet.
Public Subnet:

Earlier we discussed that public subnets allow inbound traffic from the public internet. Therefore, once you create a route in a Route Table to the IGW and associate that Route Table to a Subnet. The Subnet becomes a Public Subnet.

Elastic IP (EIP)

When a Virtual Machine is created in a Public Subnet, it can be given a Public IP address. These Public IP addresses are owned by the Cloud Provider and could change upon rebooting your VM. Changing IP addresses can be a problem if this is how we route traffic to the VM.

An Elastic IP (EIP) is a reserved static public IP address and can be kept even once the VM is rebooted, shutdown, or terminated. It can be reassigned to another VM.

Elastic IPs can be used in various scenarios, including:
  • Load balancing: Distributing traffic across multiple virtual machines using a load balancer.
  • Web servers: Providing a consistent public IP address for a web server.
  • VPN connections: Establishing VPN connections between your VPC and an on-premises network.

NAT Gateways

Understanding NAT Gateways
  • A NAT gateway is a network appliance that performs network address translation (NAT) for virtual machines within a VPC.
  • It allows virtual machines to use private IP addresses to access the internet while using a single public IP address for outbound connections.
  • NAT gateways are useful for reducing the number of public IP addresses required for a VPC.
Using an NAT Gateway
  1. Create an NAT Gateway.
  2. Attach it to a Virtual Private Cloud (VPC).
  3. Create a route in a Route Table to the NAT Gateway.
  4. Associate the Route Table to a Subnet.
Private Subnet:

Earlier we discussed that Private Subnets do not allow inbound traffic from the public internet. But in some situations servers in Private Subnets require access to download patches and updates from the public internet. By create a route to a NAT Gateway, the Private Subnet uses NAT to allow outbound communication (egress) to the public internet.

Cloud Endpoints

Cloud Endpoints provide a method to connect your cloud resources to other cloud services. By default, public endpoints are availble to all cloud services but requests travel over the public internet. Alternatively, creating private endpoints allow improved security as communication is within the private network.

Cloud Endpoint Types

There are three main types of cloud endpoints:

  • Public endpoint: Provides public access to all cloud services via the public internet.
  • Interface endpoint: Provides private access via a private IP address to specific services.
  • Gateway endpoint: Provides private access via a list of private IP ranges used by the specific services, such as Amazon S3 and Amazon DynamoDB.

VPC Peering

Understanding VPC Peering
  • VPC peering allows you to connect two VPCs within the same region.
  • This enables you to securely communicate between resources in the different VPCs without exposing them to the public internet.
  • VPC peering is useful for creating private networks between different accounts or regions.
Creating a VPC Peering Connection
  • To create a VPC peering connection, you need to specify the two VPCs that will be connected.
  • You can also configure the peering connection to allow traffic in both directions or only in one direction.
VPC Peering Benefits

VPC peering offers several benefits, including:

  • Secure communication: VPC peering allows you to securely communicate between resources in different VPCs without exposing them to the public internet.
  • Increased flexibility: VPC peering provides flexibility in how you can organize and manage your cloud resources.
  • Reduced costs: VPC peering can help you reduce costs by allowing you to share resources across different VPCs.

Private Connections

The following are 2 types of private connections to the cloud:

  • Direct Connect
  • VPN
Direct Connect
  • Direct Connect is a dedicated, private network connection between your on-premises data center and the cloud provider.
  • It provides a high-performance and low-latency connection for transferring data between your on-premises environment and your cloud resources.
  • Direct Connect is ideal for applications that require high bandwidth, low latency, or security.
VPN
  • A VPN (Virtual Private Network) is a secure, encrypted connection between your on-premises network and cloud using a public internet connection.
  • VPNs can be used to create a private tunnel between your networks, allowing you to securely access cloud resources from your on-premises environment.

There are two main types of VPNs:

Site-to-Site VPN
  • A site-to-site VPN connects your on-premises network to a VPC.
  • It requires a VPN gateway on both sides of the connection.
  • Site-to-site VPNs are suitable for connecting entire networks together.
Client VPN
  • A client VPN allows individual users to connect to a VPC from their devices.
  • It requires a client VPN endpoint in the cloud and a VPN client on the user's device.
  • Client VPNs are suitable for remote access to cloud resources.

Transit Gateways

Understanding Transit Gateway
  • A transit gateway is a central hub that allows you to connect multiple VPCs together.
  • It simplifies network connectivity and reduces the need for VPC peering connections.
  • Transit gateways are useful for creating large-scale, interconnected network topologies.
Creating a Transit Gateway
  • To create a Transit Gateway, you need to specify the region where it will be located.
  • Once created, you can attach VPCs to the Transit Gateway using Tansit Gateway Attachments.
  • A Direct Connect can be attached to the Transit Gateway using a Direct Connect Attachment.
  • A VPN Connection can be attached to the Transit Gateway using a VPN Attachment.
Transit Gateway Benefits

Transit gateways offer several benefits, including:

  • Simplified network connectivity: Transit Gateways provide a central hub for connecting multiple VPCs.
  • Reduced complexity: Transit Gateways can reduce the complexity of managing large-scale network topologies.
  • Improved scalability: Transit Gateways can be easily scaled to accommodate growing network requirements.
  • Hybrid connectivity: Transit Gateways provide hybrid connectivity to on premise networks.
